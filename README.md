# Fresh Chain WMS · 물류 관리 데모 (coldchain-demo)

메인 랜딩 사이트(`~/coldchain-wms-react`)의 **"데모 보기"에서 연결되는 실제 물류 관리 데모 페이지**입니다.
관리자 대시보드 형태로, 현재는 사이드바 **"대시보드"** 메뉴의 메인(골조) 페이지가 구현되어 있습니다.

## 실행

```bash
npm install
npm run dev      # 개발 서버 (기본 포트, 팀 컨벤션: 4700)
npm run build    # 타입체크 + 프로덕션 빌드
```

## 구조

```
src/
├─ App.tsx                    # 라우터: / (대시보드) · /:section (골조 placeholder)
├─ layouts/DemoLayout.tsx     # 다크 사이드바(데스크톱 고정/모바일 드로어) + Outlet
├─ pages/
│  ├─ DashboardPage.tsx       # ★ 대시보드 메인 (골조 완성)
│  └─ PlaceholderPage.tsx     # 나머지 메뉴 골조 슬롯 (팀원 작업 예정)
├─ data/dashboard.ts          # 목업 데이터 (추후 MSW + TanStack Query 교체 지점)
├─ components/  (icons, Reveal)
└─ hooks/       (useInView, useCountUp)
```

## 대시보드 구성 — 도메인 오버뷰

대시보드는 특정 도메인(주문)만 깊게 파는 대신, **사이드바 각 메뉴를 한눈에 요약**하는 오버뷰 구조입니다.

- **상단 KPI 6종 (도메인 교차)**: 전체 주문 / 배송 중 / 정시 배송률 / 창고 가동률 / 운행 기사 / 온도 이탈 — 카운트업 + 추세 칩
- **운영**: 주문 관리(상태 누적바+범례) · 배송 관리(배송중/완료/지연 + 정시율 게이지) · 경로 최적화(거리·연료·시간·효율 + AI 배지)
- **자원**: 창고 관리(창고별 가동률 바 + 온도 배지, 넓게) · 기사 관리(운행/대기/휴무 + 우수기사 TOP3)
- **분석 · 시스템**: 분석 리포트(주간 입·출고 막대 차트, 넓게) · 알림/메시지(콜드체인·유통기한·지연·재고)
- 각 카드에 도메인 아이콘 + "자세히 →" 링크로 해당 섹션 연결 → 오버뷰에서 상세로 자연스러운 이동
- **로딩 UX**: 진입/새로고침/기간전환 시 스켈레톤 → 데이터 등장, KPI 카운트업, 게이지·바 성장, 섹션 순차 reveal
- **반응형**: lg 미만 사이드바 → 햄버거 드로어, 카드 3열→1열·KPI 6→3→2열, `prefers-reduced-motion` 대응
- **데모 안내 배너** + 사이드바 하단 "메인 홈페이지로 이동" 링크

## 팀 작업 가이드

- 사이드바 나머지 메뉴(주문/배송/경로/창고/기사/리포트/알림/설정)는 `PlaceholderPage`가 골조로 연결되어
  있습니다. 새 페이지는 `src/pages/`에 만들고 `App.tsx` 라우트만 교체하세요.
- 스타일 컨벤션은 메인 프로젝트와 동일: sky/slate 팔레트, 카드 `rounded-2xl border-slate-200`,
  데이터 배열 → `.map`, `<Reveal>` stagger.
- 메인 사이트 연결: `DemoLayout.tsx`의 `MAIN_SITE_URL` (배포 시 실제 URL로 교체).
  메인 쪽 "데모 보기" 버튼에는 이 데모의 URL(로컬: http://localhost:4700)을 연결하면 됩니다.
